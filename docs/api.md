# API Specification

## Overview

The backend is built with **FastAPI** and serves a REST API consumed by the React + TypeScript frontend.

Supabase is used as the PostgreSQL database. Since Version 1 is a demo, all endpoints are public and do not require authentication. The API is organized so authentication can be added later without breaking existing routes.

**Base URL**

```
/api/v1
```

---

# Products

## Get All Products

**GET**

```
/products
```

### Access

Public

### Query Parameters

| Parameter | Description                   |
| --------- | ----------------------------- |
| category  | Filter by category slug       |
| search    | Product search                |
| featured  | true / false                  |
| min_price | Minimum price                 |
| max_price | Maximum price                 |
| sort      | newest, price_asc, price_desc |

### Response

Returns a paginated list of products with:

* id
* name
* slug
* thumbnail
* price
* category
* featured

---

# Cart

Cart operations are linked to an anonymous **session_id** generated by the frontend.

---

## Get Cart

**GET**

```
/cart/{session_id}
```

### Access

Public

Returns:

* Cart items
* Quantities
* Prices
* Subtotal

---

## Add Item

**POST**

```
/cart
```

### Access

Public

Request Body

```json
{
  "session_id": "string",
  "product_id": "uuid",
  "size": "L",
  "quantity": 1
}
```

---

## Update Quantity

**PATCH**

```
/cart/{item_id}
```

### Access

Public

Request Body

```json
{
  "quantity": 2
}
```

---

## Remove Item

**DELETE**

```
/cart/{item_id}
```

### Access

Public

Deletes a cart item.

# Orders

## Submit Demo Order

**POST**

```
/orders
```

### Access

Public

Request Body

```json
{
  "session_id": "string",
  "customer_name": "John Doe",
  "email": "john@example.com",
  "phone": "+91XXXXXXXXXX",
  "address": "Customer Address"
}
```

### Response

```json
{
  "success": true,
  "message": "Demo order submitted successfully."
}
```

The backend creates an order record and associated order items. No payment or shipping workflow is triggered.

---

## Get Demo Order

**GET**

```
/orders/{order_id}
```

### Access

Public

Returns the submitted order summary for display on the confirmation page.

# Authentication

**Version 1**

* No authentication required.
* Anonymous carts are managed with a frontend-generated `session_id`.

**Future Version**

Authentication can be added using **Supabase Auth** with minimal API changes. Protected endpoints (e.g., user profiles, order history, wishlist) would then require JWT-based authorization.

---

# Error Response Format

All endpoints should return errors in a consistent structure:

```json
{
  "success": false,
  "error": {
    "code": "RESOURCE_NOT_FOUND",
    "message": "The requested product could not be found."
  }
}
```

# API Design Principles

* RESTful endpoint naming.
* UUIDs for primary resources.
* JSON request and response bodies.
* Proper HTTP status codes (`200`, `201`, `400`, `404`, `422`, `500`).
* Pagination support for product listings.
* Validation handled using FastAPI with Pydantic models.
* CORS configured to allow requests from the React frontend.
* Clear separation of routers, services, schemas, and database access layers to keep the codebase maintainable as the project grows.
